    V2.12
    ------
    Updates:
        - Added new options.
          - New options: R_FCL_NO_FCU_COPY, R_FCL_MIRROR_FCU_COPY, R_FCL_CFL2_START_ADDR_2048K

    V2.11
    ------
    Updates:
        - Added F1K device group support
          - Changed the FCU RAM transfer
          - Added Code flash buffer control
          - Changed the pre-fetch size of R_FCL_CODE_RAM_EX_PROT section
          - Changed the Makefile of the sample program
            Added the "-Xcpu = G3K" in the CC-RH compiler options.
            Common object file for the RH850 devices including the G3K is 
            generated by this option.
        - Added timeout supervision
          - Added timeout supervision for library internal loops checking 
            for HW status changes.
    Improvement:
        - Added measure to avoid unwanted error signalling
          - Added measure to avoid unwanted error signalling to an ECM during 
            FACI initialization (Only devices with ECM).
        - Misc        
          - Minor updates for the maintenance of the library
          - Added "-Xreserve_r2" to the compiler options of the CC-RH sample 
            program makefile. 
            This was done in order to support RTOS [RI850V4] V2 for RH850.
          - Delete the unnecessary "-Xreserve_r5" of GHS compiler options 
            in the Makefile of the sample program.
          - Minor updates for the maintenance of the sample program
    Fixes:
        - Added SYNCP instruction before SYNCI.
          - Solved the issue of incorrect description of synchronization 
            procedure in "User's Manual:Hardware" of each product.
        - Modified the R_FCL_Fct_PrgOffset in R_FCL_CalcFctAddr function 
          for only CC-RH.
          - Modified the assembler instruction,because a failure is caused by 
            the sign-extended bits.
    Support devices:
        - This version supports F1x(L, M, H, K), D1x(L, M). 
        - Customers using other than above device groups:
          Please contact Renesas regarding Code and Data Flash Self-Programming 
          support of your devices. Refer to the user's manual of this library 
          (chapter 1) for Renesas contact.

    V2.00
    ------
    Major version update due to functionality update and moderate API changes

    Updates:
        - Added cancel request for the Flash operations Erase and Write
        - Support of devices equipped with multi-core architectures 
          (with Hardware Security Module). 
          - Resulting in slightly increased number of handler calls required 
            for Flash operations
        - Added robustness checks
          - Prepare environment operation is now prohibited on active Flash 
            operations
        - Further minor internal improvements and updates
    Fixes:
        - Fixed issues in suspend/resume processing:
          - If after suspending a Flash operation a new Flash operation is 
            executed to another Flash area (e.g. Erase to user boot area suspended 
            --> write to user area), the resumed operation continues on 
            the wrong area
          - Issues in case of a short transition periods
            1) 
              - Conditions
                - Suspend request when a Flash hardware operation is not yet 
                  started, as not yet handled by R_FCL_Handler
              - Issues
                - The status of the suspended operation is possibly not 
                  properly updated if another Flash operation is executed when 
                  the 1st one is suspended
            2) 
              - Conditions
                - One affected dedicated sequence: 
                  Flash operation start - suspend request - hdr call - resume 
              - Issues
                - No further Flash operation is possible
    API changes:
        - Suspend / Resume requests
          - Old: 
            Suspend request was accepted but ignored for other commands than 
            Erase and Write. This resulted in long suspend time for other 
            commands
          - New:
            Suspend request is only accepted for Erase and Write commands. For
            other commands it is now rejected
        - R_FCL_Init returned value "R_FCL_ERR_PROTECTION" moved to the Prepare
          environment command handling
        - Prepare environment command only executable once after R_FCL_Init 
          execution
          - Old: Execution possible without restriction after R_FCL_Init

    Added F1H ,F1M support

======================
2. General Information
======================

The code Flash Library is delivered in source code, prepared for
the GHS development environment, IAR embedded workbench and the 
Renesas CS+.

Supported compiler are:

- GHS 6.16, compiler version 201517
- Renesas CC-RH, compiler version 1.03.xx
- IAR embedded workbench 1.30.x
Please contact Renesas for information on the development environment patch versions used for testing.

This installer contains the Code Flash library for the different
devices mentioned in support.txt. By selecting the correct device and
required compiler the appropriate libraries will be installed automatically.

Sample applications in this installer package are build using GNU Make for
Windows Version 3.81.
To build a sample application using the provided batch files, please install
GNU Make from the internet, and adapt the batch files accordingly.
Further more, please adapt also the path to your compiler installation in the
"Makefile" of the sample application.

3. File Naming
========================================

The installer file has the name RENESAS_FCL_RH850_T{ab}_V{x}.{y}{z}.exe
e.g. RENESAS_FCL_RH850_T01_V1.00.exe

{ab} Library type number
{x} This version digit will be incremented in case of major changes e.g. API
changes
{y} This version digit will be incremented in case of bug fixes and internal
improvements
{z} This version digit will be incremented in case of adding support of new
devices

========================
4. Operating Precautions
========================

All devices and libraries
-------------------------
- The FCL version V2.11 or later requires that The descriptor is placed in RAM.
- The following warnings are issued during building in the Renesas version 
  FCL V1.02 or later (product version), but they do not indicate any 
  operational problems.
    W0561322:Section alignment mismatch: "R_FCL_CODE_USR.text"
    W0561322:Section alignment mismatch: "R_FCL_CODE_RAM.text" 
    W0561322:Section alignment mismatch: "R_FCL_CODE_ROMRAM.text"
  Warning messages can be controlled by the CS+ option setting. 
  For details, see the CS+ User's Manual: CC-RH Build Tool Operation.
- F1x and D1M1A have not user boot area but extended user area .
  Currently, F1x and D1M1A support the programming to extended user area.
  Programming the user boot area of all device are not supported as a specification.
- The libraries partially have been tested on not qualified hardware (working sample).
  Following that, it might be necessary to update the libraries later on in 
  case of changing device features.
- For further information, please refer to the related documentation and device documentation.
